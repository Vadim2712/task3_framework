# Веб-служба с расширенной конфигурацией и защитой

Этот проект представляет собой легковесную веб-службу на Node.js, разработанную для демонстрации лучших практик по управлению конфигурацией и обеспечению базовой безопасности API.

## Ключевые возможности

- **Гибкая конфигурация**: Чтение настроек из файла, переменных окружения и аргументов командной строки.
- **Приоритет настроек**: Четко определенный порядок приоритета: аргументы командной строки > переменные окружения > файл `appsettings.json`.
- **Ранняя валидация**: Проверка корректности всех критичных параметров перед запуском. Сервер не запустится с неверной конфигурацией.
- **Два режима работы**:
  - `учебный`: Вывод подробных сообщений об ошибках.
  - `боевой`: Краткие и безопасные сообщения об ошибках.
- **Безопасность**:
  - **CORS**: Разрешает запросы только с доверенных доменов.
  - **Rate Limiter**: Ограничивает частоту запросов по IP-адресу с разными лимитами для чтения и записи.
  - **Защитные заголовки**: Автоматически добавляет заголовки `X-Content-Type-Options`, `X-Frame-Options` и `Cache-Control` для защиты от базовых атак.

## Требования

- [Node.js](https://nodejs.org/) (рекомендуется версия 18.x или выше)

## Установка

1.  Склонируйте репозиторий:
    ```sh
    git clone <адрес_репозитория>
    cd task3_framework/node
    ```
2.  Установите зависимости (хотя в проекте их нет, это хорошая практика):
    ```sh
    npm install
    ```

## Запуск

### Стандартный запуск

Для запуска сервера в режиме, указанном в файле `config/appsettings.json`, выполните:

```sh
npm start
```

По умолчанию сервер запустится на порту `3000` в режиме `учебный`.

### Запуск с кастомными настройками

Вы можете переопределить любые параметры при запуске.

**Через переменные окружения:**

```sh
# Запуск в боевом режиме на порту 4000
APP_MODE=боевой APP_PORT=4000 npm start
```

**Через аргументы командной строки (имеют наивысший приоритет):**

```sh
# Запуск с другим лимитом на чтение и доверенным источником
npm start -- --port=5000 --readPerMinute=100 --trustedOrigins=http://my-app.com
```

## Тестирование

Для запуска встроенных тестов, которые проверяют логику конфигурации и безопасности, выполните:

```sh
npm test
```

## Конфигурация

### Источники настроек и приоритет

1.  **Аргументы командной строки**: (например, `--port=8080`) - **Высший приоритет**.
2.  **Переменные окружения**: (например, `APP_PORT=8080`).
3.  **Файл `config/appsettings.json`**: Значения по умолчанию - **Низший приоритет**.

### Параметры

- `mode`: Режим работы (`учебный` или `боевой`).
- `port`: Порт для запуска сервера.
- `trustedOrigins`: Список разрешенных источников (через запятую), например `http://localhost:5173,https://my-site.com`.
- `rateLimits.readPerMinute`: Максимальное количество GET-запросов в минуту.
- `rateLimits.writePerMinute`: Максимальное количество POST-запросов в минуту.
